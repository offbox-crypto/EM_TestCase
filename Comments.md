## Задание 2
Не совсем понятно зачем отображать время доставки, если ссылка на корень сайта партенера.<br>
Теоретически, перед формированием экрана необходимо проверить доступность ресурса, наличие доставки и т.п. <br>
Данные берутся с собственного сайта greenpars(Петрушка Зеленая).com из xml файла, или таблицы ("Partners") выполнением скрипта на сервере с обращением в БД. <br>
Обращение к источнику напрямую думаю - не безопастно.<br>
Занчения таблицы(файла) обновляются (м.б. в момент обращения или постоянно, в фоновом режиме) скриптом обращения к источникам (ресурсы партнеров).<br>
Таким образом таблица(файл) всегда содержит актуальные данные.<br>
Не понял зачем оговорка "Учесть, что при клике на плашку магазина должен осуществляться переход по ссылке на внешний ресурс"..<br>
Если бы имел место переход к сайту партнера с конкретным продуктом или другим параметром(например, время доставки) то, следовало бы усложнить целевую ссылку.<br>
Таблица(файл) содержит поля:
 - Id - ID
 - Name - Название партнера
 - Status  - Активный, Не активный 
 - Delivery - Осуществляет доставку или нет
 - DeliveryType  - Тип доставки (Стандартная, Экспресс)
 - DeliveryTime - времена доставки
 - PertnerURL - URL Партнера
 - PartnerLogo - URL логотипа партнера для кнопки.
Далее рапарсить json и отобразить только активных партнеров, со вшитыми ссылками для перехода.
Все очень упрощенно. Еще разных много нюансов.
